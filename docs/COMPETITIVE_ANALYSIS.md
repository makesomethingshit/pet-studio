# 경쟁 분석: 왜 기존 AI 도구는 비전공자에게 어려운가

## 기존 도구들이 어려운 이유

### Cursor
- **코드 에디터가 전제** — 비전공자는 코드 에디터 자체를 무서워함
- 터미널, 파일 탐색기, 디버그 콘솔이 기본 화면에 노출됨
- "AI가 코드를 짜준다"는 건 좋지만, 비전공자는 코드를 읽을 줄 모름
- 결과물을 확인하려면 코드를 열어봐야 함

### Claude Desktop
- **채팅창이 전부** — 프로젝트 관리, 팀 상태, 작업 배분이 안 보임
- 대화 기록이 길어지면 이전 맥락을 찾기 어려움
- "지금 AI가 뭘 하는지" 시각적으로 확인할 수 없음
- 여러 작업을 동시에 진행하면 대화가 뒤섞임

### OpenClaw
- **설정 파일을 직접 편집** — YAML/JSON 파일을 열어서 수동으로 설정
- 커맨드라인 기반 설치 및 실행
- 에러 메시지가 기술적 (스택 트레이스, 로그 파일 경로 등)
- 커뮤니티가 기술 중심이라 비전공자 질문에 대한 답변이 부족

## Pet Studio가 다른 점

| 기존 도구 | Pet Studio |
|---|---|
| 코드 에미터 기반 | 미션 입력 + 상태 확인이 전부 |
| 채팅창 기반 | 프로젝트/팀/태스크가 시각적으로 보임 |
| 설정 파일 편집 | UI에서 전부 가능 |
| 로그를 뒤져야 확인 | 한 창에서 바로 보임 |
| 기술 에러 메시지 | 사용자 친화적 토스트/상태 표시 |

## 핵심 차별점

**비전공자는 "AI가 뭘 하는지" 보고 싶은 거지, "AI가 어떻게 동작하는지" 알고 싶은 게 아니다.**

- Cursor는 "어떻게 동작하는지"를 보여줌 (코드, 터미널, 디버그)
- Pet Studio는 "뭘 하는지"를 보여줌 (프로젝트 상태, 팀 역할, 태스크 진행)

이 차이가 핵심이다.
