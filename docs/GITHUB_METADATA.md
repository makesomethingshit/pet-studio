# GitHub Metadata Suggestions

Use this when setting the repository About section manually on GitHub.

## Recommended Description

```text
Turn each Codex project into a tiny layered desktop room with pets, props, and live state bubbles.
```

```text
A tiny desktop room for every Codex project.
```


## Suggested Website / Link

If the repo has no project page yet, leave this blank. A short demo GIF in the README is more important than a placeholder website.

## Pinning Note

Pin a release screenshot or GIF near the top of the README before doing a larger launch. The project is visual, so the About metadata works best when the first README screen already shows the room.
