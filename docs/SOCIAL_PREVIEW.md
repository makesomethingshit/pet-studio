# Social Preview Guidance

The GitHub social preview should communicate the whole project without needing the README.

Recommended final text:

```text
Every project gets a room.
```

## Ideal Image

- A small pixel-art or sticker-style room.
- A pet in the center or left-center.
- A speech bubble saying `Using shell...` or `Working...`.
- Two or three smaller project room cards in the background.
- Main text:

```text
Every project gets a room.
```

## Tone

Playful, local, and developer-focused. It should feel like a tiny dashboard, not a generic mascot logo.

## Safe Asset Rules

- Do not use copyrighted game art, anime frames, or unlicensed character art.
- Prefer generated or repo-owned art.
- If using a screenshot, make sure no private project names, paths, tokens, or chat content are visible.

## Placeholder

A simple repo-safe SVG placeholder is available at:

```text
assets/social-preview.svg
```

It is intentionally plain and safe to use for the first public pass. Replace it with a real screenshot or GIF frame before a bigger launch if possible.

## GitHub Upload Steps

1. Open the repository settings.
2. Go to Social preview.
3. Upload a `1280x640` image.
4. Prefer a real widget frame if it clearly shows the room, pet, and bubble.
5. Use `assets/social-preview.svg` only if no real frame is ready.
