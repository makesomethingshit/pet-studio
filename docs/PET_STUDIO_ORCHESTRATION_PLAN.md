## 에이전트 구성

| 구분 | 인스턴스 | 엔드포인트 | 역할 |
|------|---------|-----------|------|
| **알바생** | 공유 전환 | 로컬 LLM (vllm/Ollama/llama.cpp) + 순수 스크립트 fallback | 상태 관리, 이벤트 감지, 큐 관리, 프로젝트 순환 감시 |
| **직원** | 풀 기반 | OpenRouter API | 프로젝트별 배정, 작업 수행, 완료 후 풀 복귀 |
| **리드** | 프로젝트별 | 유저 선택 (Codex/Claude Code/Cursor/Continue 등) | 고가 판단, 구현, 최종 결정 |

### 알바생 상세
- GPU 있는 로컬에서만 활성 (전기세 문제)
- 없으면 순수 스크립트 모드로 동작 (감지만 하고 판단은 안 함)
- 프로젝트 큐잉: 활성 프로젝트 우선 → 비활성 프로젝트 순환 감시
- 엔드포인트: vllm (기본) / Ollama / llama.cpp / script

### 직원 상세
- OpenRouter API 기반, GPU 부담 없음
- 프로젝트마다 적합한 모델 선택 (코딩 → 코딩 특화, 문서 → 범용)
- 풀 기반: 빈 직원에게 작업 배분 → 완료 → 풀 복귀
- 동시 여러 프로젝트 작업 가능

### 리드 상세
- 유저가 선택 (Codex 기본, Claude Code/Cursor/Continue 등 확장)
- 프로젝트마다 다른 리드 배정 가능
- 가장 비싼 리소스 → 큐잉 필수

## 권한/스킬 체계

### 역할 기본 템플릿 + 세부 조정

```
알바생 → 감시 팩 (읽기, 상태관리, 이벤트감지)
직원   → 코딩 팩 (읽기, 쓰기, 코드생성, 검색) + 커스텀 조정
리드   → 전체 팩 (전체 툴) + 보안 레벨에 따라 제한
```

유저는 기본 템플릿을 선택하고, 필요시 개별 스킬 추가/제거 가능.

### 보안 레벨

```
L0 허용  → 리드 자체 판단, 모든 작업 허용
L1 경고  → 위험 작업 시 로그만 남기고 실행 (기본값)
L2 승인  → 위험 작업 시 유저 승인 필요
L3 잠금  → 위험 작업 완전 차단
```

프로젝트 설정에서 변경. 기본값 L1.

### 스킬 팩

```
기본 스킬 팩:
  알바생 → "감시 팩" (읽기, 상태관리, 이벤트감지)
  직원   → "코딩 팩" (읽기, 쓰기, 코드생성, 검색)
  리드   → "전체 팩" (전체 툴)

유저 커스텀:
  개별 스킬 추가/제거
  커스텀 스킬 팩 저장/공유
```

## 프로젝트 전환

```
1. 워크스페이스 감지 → 활성 프로젝트 자동 전환
2. 비활성 프로젝트 → 알바생 순환 감시 (큐잉)
3. 직원/리드 작업 → 큐에 쌓여서 순서대로 처리

큐 시스템:
  Project A: [이벤트3, 이벤트2, 이벤트1] ← 알바생 처리중
  Project B: [이벤트1] ← 대기
  Project C: [] ← 유휴
```

## 유저 인터랙션

```
위젯: 상태 확인, 프로젝트 전환, 팀 관리 (클릭/드롭다운)
채팅: 복잡한 미션 지시, 팀 회의, 세부 조정 (자연어)
단축키: 프로젝트 전환, 팀 패널 열기 (자주 쓰는 명령)
```

## 점진적 똑똘함 (Progressive Intelligence)

```
0.5.0 — 컨텍스트 축적
  팀 상태 파일에 작업 이력 누적
  알바생이 이력 기반으로 이벤트 분류 정확도 향상

0.6.0 — 패턴 학습
  유저 반복 작업 감지 → 자동화 제안
  "매번 커밋 전에 린트 돌리시네요. 자동화할까요?"

0.7.0 — 프로액티브
  이슈 감지 시 유저에게 미리 알림
  "이 파일 자주 터치하는데 에러 로그 늘고 있어요"

0.8.0 — 팀 협업 최적화
  어떤 직원 조합이 잘 되는지 학습
  실패한 작업 → 다음에 다른 직원에게 배정

0.9.0 — 자기 개선
  실패 이력에서 패턴 추출
  "이 유형의 작업은 직원A보다 직원B가 성공률 30% 높음"
```
